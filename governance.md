# 站長營運指南

本指南是給 metawilo.com 站長的營運建議：如何在沒有專業群眾外包專案經驗的情況下，把這個專案經營得既有公信力、又不會把自己累垮、也不會跨過「問責」與「肉搜」之間的那條線。

內容分五個面向：管理結構、台灣可學專案、國際參考、UI 設計模式、風險管控紅線，最後給一個本週可以做的高槓桿動作。

---

## 1. 一人創辦人的務實管理

擴大規模之前，三件事最重要。

### 單一資料源 + 單一協作管道

用一個 GitHub repo 涵蓋全部：

- `records/` 放 JSON 紀錄。
- Pull Request 做品管審查。
- Issues 處理下架與更正請求。
- Discussions 開放一般問題。
- GitHub Pages 直接做網站托管。

**不要把資料、討論、提交分散到 Discord + email + 雲端硬碟 + 表單**，站長會在三個月內燒掉。Repo 內建稽核軌跡、PR 流程強制品質、所有歷史一鍵可查。

### 信任層級制度

一個人沒辦法永遠逐筆審查。建立信任分級：

| 層級 | 條件 | 審查方式 |
|---|---|---|
| L0 新貢獻者 | 剛加入 | 每筆紀錄逐行人工審查 |
| L1 | 累計約 10 筆無誤 | 隨機抽樣審查 |
| L2 | 累計約 50 筆無誤 | 可預審其他 L0 的 PR |
| L3 共同維運 | 受邀 | 擁有 merge 權限 |

**公開晉升條件**，避免「靠關係升等」的質疑。

### 決策日誌

站長每次的判斷（例：「更生人滿 10 年不收」、「酒駕初犯不收」、「某類案件納入」），都寫成 `decisions.md` 中的一條編號紀錄。

好處：

- 同樣的爭議不需要重新討論。
- 對外質疑「為什麼這樣處理」時有可引用的依據。
- Wikipedia、Bellingcat、OCCRP 都用這個模式。

### 例行衛生

- 設定**不公開**的 `legal@metawilo.com`（或類似信箱）專門處理下架／法律威脅。**不要**用公開頻道回應。
- 公布版本化的「更正與下架政策」單頁文件，建立可防禦的程序基礎。
- 對外開放前，找律師做一次性審查（任務範圍：站務宣言 + 下架政策 + 個資法合規）。台幣幾千到一萬，是最划算的保險。
- **不要急著紅。** 先用 5–10 位信任的志工，把標準磨好，再擴大徵召。
- PR 審查改成**每週一次集中處理**，不要即時回應，保護站長身心。

## 2. 可學習（或合作）的台灣專案

- **g0v 零時政府** 是要插入的生態圈。
  - 兩個月一次的大松（hackathon）可以去 demo 這個專案，向跑了十年公民資料專案的人請益。
  - 他們累積了大量「群眾資料專案治理」的實作智慧。
  - 可能找到共同維運人。
- **g0v 政商人物關係資料庫（政誌）**：最接近的兄弟專案，蒐集公開的政商人物資訊與關係。研究他們的方法論頁面與來源標註。
- **法律白話文運動**：已經在做白話文法律內容、有編輯標準與公信力。可能是合作夥伴或寫作風格參考。
- **法操 FollowLaw**：司改觀察站，他們的案件摘要卡片 UI 非常適合一般讀者。
- **沃草 Watchout**：問責新聞 + 資料庫，編輯標準可以參考。
- **Cofacts / 美玉姨**：群眾事實查核，他們的「驗證再公布」迴圈值得學。

最高槓桿的單一動作：去下一次 g0v 大松發表這個專案，**十年公民資料專案經驗免費換給你**。

## 3. 國際參考

### 治理（如何維持問責性而不淪為肉搜）

- **Wikipedia 在世人物傳記政策（Biographies of Living Persons, BLP）**：黃金標準。涵蓋知名度、可驗證性、中立觀點、對無來源負面陳述的即時移除義務。建議幾乎逐條引用、本土化。
- **OCCRP（國際組織犯罪與貪腐報導計畫）**：嚴格來源標準，從不刊登未經驗證資訊。
- **OpenCorporates**：公開紀錄聚合的透明度典範。
- **Free Law Project / CourtListener**：美國法院紀錄資料庫 + NLP 摘要。他們的爭議處理文件值得讀。

### 警世案例（要避免變成什麼）

- **美國各州性侵犯公開登記**：引發訴訟與私刑文化。教訓：嚴控範圍、明確公益框架、健全下架 SLA。
- **「犯罪紀錄聚合」SEO 站**：許多最終變質為「付費才下架」的勒索生意。
  - **絕對不要對下架收費。**
  - **絕對不要對個別紀錄賣廣告。**

## 4. UI / 資料美化的設計模式

讓法律與公開紀錄資料對一般人可讀的成功模式：

- **個別當事人頁**：照片（含來源標註）+ 案件時間軸 + 白話文摘要卡 + **每一項主張都附原始判決書連結**。
- **每一個事實的內嵌來源連結。** 這是建立公信力最大的槓桿。
- **每筆紀錄附「最後驗證日期」。**
- **可篩選索引**：依罪名、年度、法院、終審結果分類。
- **聚合 / 趨勢頁**：依年度與罪名類型呈現匿名統計，吸引讀者而不聚焦在個人。
- **方法論頁面**：紀錄如何被納入、如何被移除、由誰審查。**這是面對批評時最重要的防線頁面。**

可借鏡的 UI：

- **TheyWorkForYou**（英國國會議員頁面）
- **CourtListener** 案件頁
- **法操** 案件摘要卡
- **Bellingcat** 調查報導頁（內嵌引註作為設計骨幹）

### 技術選擇建議

**靜態網站**（Astro / Hugo / Eleventy）把 repo 的 JSON 紀錄轉成 HTML：

- 免費 hosting（GitHub Pages 或 Cloudflare Pages）。
- 無需資料庫維護。
- 容易鏡像備份。
- [schema.md](./schema.md) 設計的 JSON 格式可以直接餵進去。

## 5. 區別「問責」與「肉搜」的六道防線

這六條合在一起，就是可信問責網站 vs 私刑工具之間的差別：

1. 每一項主張都有**內嵌來源**。
2. 公開且有 SLA 的下架／更正流程（建議 7 個工作天內初步回應）。
3. 公開**透明度日誌**：何時新增、移除、修改了哪些紀錄，原因為何。
4. **使命框架**定位為「公開、確定司法結果的公益認知」，不是「揭露犯人」。
5. 只收**確定有罪判決**；排除清單公開鏡像在網站上。
6. **不從個別紀錄營利**：不對下架收費、不對個別紀錄賣廣告、不轉售資料。

## 6. 本週可以做的一個高槓桿動作

找 g0v 下一次大松的時間（通常隔月週六），登記發表 5 分鐘的專案介紹。十年公民資料專案的智慧免費換給你，可能還順便找到共同維運人或律師資源。

## 延伸：給站長的最小起步清單

如果還沒準備好做這麼多，這是最小可行版本，按順序執行：

1. 開一個 GitHub repo（公開或私有都可以），把 [`metawilo-contrib/`](./) 整個放進去。
2. 開 `records/` 空目錄，加一個 `.gitkeep`。
3. 把 `submission.md` 中的「站長承諾事項」與「更正與下架政策」單獨抄成根目錄的 `POLICY.md`，**逐字審視後決定要不要採用**。
4. 加一個 `decisions.md`，未來每個判斷寫一條編號。
5. 自己跑一遍 [`workflow.md`](./workflow.md) 的 8 個步驟，產出一筆紀錄 commit 進 `records/`，當示範。
6. 找 2–3 位信任的朋友當 L0 試跑，**至少跑滿 4 週**再考慮公開徵募。
7. 加入 g0v Slack（jothon.g0v.tw），自我介紹專案。
