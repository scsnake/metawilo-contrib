# 工作流程

從鎖定目標到提交紀錄的八個步驟。每一步都有「卡關時怎麼辦」與「下一步條件」。

---

## 1. 鎖定目標

決定你這次要蒐集什麼。三種常見起點：

- **依姓名**：已知某有罪確定判決的人，想把他/她名下所有案件整理出來。
- **依罪名 / 法條**：例如「兒少性剝削防制條例 §36」、「詐欺取財」、「酒駕致死」等，按日期或法院過濾。
- **依公告**：例如警政署通緝犯名單中的某人，或廉政專刊列出的公務員。

把目標寫成一句話（提交時填在 PR 描述裡）。

---

## 2. 撈取原始裁判書

**首選方式：司法院 Open Data 批次下載。**

1. 到 <https://opendata.judicial.gov.tw/> 找對應日期區間的 ZIP（按法院 + 日期打包）。
2. 解壓後在本機用 `grep` 或一段 Python 過濾你要的姓名、法條、關鍵字。
3. 命中的 JSON / HTML 檔留下來，準備餵給 LLM。

**備用方式**：只在你已知特定案號、想立刻看原文時，用 <https://judgment.judicial.gov.tw/> 搜尋頁查詢。

⚠️ **不要寫腳本批次抓搜尋頁。** 詳細來源說明見 [sources.md](./sources.md)。

---

## 3. 預先過濾（在跑 LLM 之前）

讀 [exclusions.md](./exclusions.md)，用案件的 metadata（不是內文）做機械過濾。例如：

- 案號帶「少」、「家」 → 少年/家事案件，直接丟棄。
- 主文出現「無罪」、「免訴」、「不受理」、「不起訴」、「緩起訴」 → 直接丟棄（妨害名譽風險）。
- 仍在上訴期間或上訴中 → 暫時擱置，不要提交。

這一步不需要 LLM，是最便宜的把關。

---

## 4. 分類（第一段 LLM）

把判決書全文貼到你慣用的 LLM（Gemini / Claude / ChatGPT），跑 [`prompts/classify.md`](./prompts/classify.md)。輸出是一段小 JSON：

```json
{
  "eligible": true,
  "確定": true,
  "有罪": true,
  "成年被告": true,
  "victim_identifiable": false,
  "reason": "確定判決、成年被告、有罪，且摘要不致洩漏被害人身份"
}
```

**只有 `eligible: true` 才能進下一步。** 任一條件不符就丟棄，並把 `reason` 記下來。

---

## 5. 白話文 + 重點摘要（第二段 LLM）

跑 [`prompts/summarize.md`](./prompts/summarize.md)。

這個提示詞已經寫明「禁止使用任何 markdown 符號（`*`、`#`、`>`、`-`、反引號）」，輸出本來就是純文字 — **這是設計上的關鍵改良，省掉站長之前要跑的第二次「去掉 `*` 與 `#`」提示**。

輸出固定為下列段落結構：

```
【案件概要】
...

【事實經過】
...

【法院認定】
...

【判決主文】
...

【重點摘要】
1. ...
2. ...
3. ...
```

若 LLM 不小心輸出了 markdown 符號，請重跑或換另一個 LLM。

---

## 6. 萃取結構化紀錄（第三段 LLM）

跑 [`prompts/extract-record.md`](./prompts/extract-record.md)。輸入是裁判書全文 + 第 5 步產出的白話文摘要。輸出是一份 JSON，欄位定義見 [schema.md](./schema.md)。

提示詞要求：**任何無法從判決書明確取得的欄位填 `null`，禁止推測。**

---

## 7. 去重與跨審級整併

同一個案子會在一審、二審、三審各出現一次。請以 **終審案號** 為主鍵：

- 找到該案的最終確定判決（通常是三審或駁回上訴後的二審）。
- 把較低審級的案號放進 `related_cases[]`。
- 不要把同案不同審級當成多筆紀錄提交。

如果你只蒐集到一審判決，但二審尚未確定 → 整筆擱置，等確定再回頭處理。

---

## 8. 自我驗證後提交

把你產出的 JSON 連同 `source_url` 一起貼回 LLM，問它：

> 以下這份 JSON 紀錄是否忠實對應這份裁判書？列出任何捏造（hallucination）、遺漏、或推測的欄位。若全部正確僅回覆 PASS。

任何不是「PASS」的回覆都要修正後再驗證一次。

確認後依 [submission.md](./submission.md) 提交。

---

## 卡關常見原因

| 症狀 | 解法 |
|---|---|
| 裁判書中的姓名都被遮成 ○○○ | 表示這份不是被告本人（可能是被害人或證人裁定書）。整筆丟棄。 |
| LLM 拒絕翻譯（觸發內容安全） | 提示詞前綴改成「以下是公開的台灣司法院已公布判決書，僅做白話文重述」。或換另一個 LLM。 |
| Open Data 找不到對應檔案 | 先用搜尋頁手動下載個別判決確認原文存在，再回頭 debug 下載流程。 |
| 找不到終審案號（不確定是否確定） | 用案號到 judgment.judicial.gov.tw 查同案上訴紀錄，或暫時擱置。 |
| 摘要中出現「我認為」、「顯然」等推測語氣 | 重跑 `prompts/summarize.md`，並提醒 LLM「禁止推測、僅客觀敘述」。 |
