# 紀錄格式 schema

每一筆案件存為一份 JSON 檔。檔名規則：`<court>_<終審案號>.json`，例如 `TPHM_113上訴1234.json`。

---

## 範例

```json
{
  "defendant_name": "王小明",
  "終審案號": "113年度上訴字第1234號",
  "審級": "二審",
  "是否確定": true,
  "判決日": "2024-08-15",
  "犯罪事實發生日": "2023-03-10",
  "法院": "臺灣高等法院",
  "起訴法條": ["刑法 §339-4 第1項第1款"],
  "罪名": ["三人以上共同犯詐欺取財罪"],
  "主刑": "有期徒刑 1 年 6 月",
  "從刑": ["褫奪公權 2 年"],
  "易科罰金": null,
  "緩刑": null,
  "共犯案號": ["113年度上訴字第1235號"],
  "related_cases": [
    {
      "審級": "一審",
      "案號": "112年度訴字第567號",
      "法院": "臺灣臺北地方法院",
      "判決日": "2024-02-20"
    }
  ],
  "重點摘要": "被告於民國 112 年 3 月聯合其他兩名共犯，假冒檢察官名義……（純文字，無 markdown）",
  "source_url": "https://judgment.judicial.gov.tw/FJUD/data.aspx?ty=JD&id=TPHM,113,...",
  "collected_by": "your-handle",
  "collected_at": "2026-06-14"
}
```

---

## 欄位說明

| 欄位 | 型別 | 必填 | 說明 |
|---|---|---|---|
| `defendant_name` | string | ✅ | 被告全名。若全文中被遮蔽成 ○○○，整筆放棄。 |
| `終審案號` | string | ✅ | 主鍵；存檔名、去重以此為準。 |
| `審級` | string | ✅ | 取值：`一審` / `二審` / `三審` / `更審`。 |
| `是否確定` | bool | ✅ | **必須為 `true` 才能提交。** |
| `判決日` | ISO date | ✅ | 終審判決日期。 |
| `犯罪事實發生日` | string | ✅ | 與判決日不同。若為期間用 `"2023-03 至 2023-06"`。 |
| `法院` | string | ✅ | 終審法院名稱。 |
| `起訴法條` | string[] | ✅ | 具體條項款，例 `"刑法 §339-4 第1項第1款"`，不要只寫「刑法 §339-4」。 |
| `罪名` | string[] | ✅ | 判決主文記載的罪名。 |
| `主刑` | string | ✅ | 有期徒刑、無期徒刑、罰金、拘役等。 |
| `從刑` | string[] | 選填 | 沒收、褫奪公權等。 |
| `易科罰金` | string \| null | 選填 | 若有，記下換算（例 `"以新台幣 1000 元折算 1 日"`）。 |
| `緩刑` | object \| null | 選填 | `{ "年數": 2, "條件": "..." }` 或 `null`。 |
| `共犯案號` | string[] | 選填 | 同案其他被告的案號。 |
| `related_cases` | object[] | ✅ | 較低審級案號軌跡。每筆 `{ "審級", "案號", "法院", "判決日" }`。 |
| `重點摘要` | string | ✅ | 第 5 步產出的白話文摘要全文，純文字無 markdown。 |
| `source_url` | string | ✅ | judgment.judicial.gov.tw 的連結。 |
| `collected_by` | string | ✅ | 你的代號或 GitHub handle。 |
| `collected_at` | ISO date | ✅ | 你產出這份紀錄的日期。 |

---

## 為什麼不收這些欄位

- **前科**：單一判決書通常不會完整記載前科；推測有妨害名譽風險。
- **加重 / 減輕事由**：放在 `重點摘要` 散文敘述裡即可，列成結構欄位反而徒增爭議。
- **照片**：若是通緝犯照片，著作權屬警政署，由站長集中處理引用，不放在貢獻者 JSON 裡。

---

## 命名與型別注意事項

- JSON 欄位名稱中的英文（`defendant_name`、`related_cases` 等）保持英文，不要翻成中文 key。其餘 key 為中文（`終審案號`、`是否確定` 等）以便閱讀。
- 日期一律 ISO 8601（YYYY-MM-DD）；民國年要換算為西元年。
- 若某欄位「裁判書沒提到」，填 `null` 而不是空字串 `""`。
- 字串內容為**繁體中文**。
