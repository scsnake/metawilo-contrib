# 白話文摘要提示詞（第二段 LLM）

用途：把裁判書翻譯成一般人能讀懂的白話文 + 重點摘要。

**這份提示詞已內建「禁止 markdown」規則**，輸出本來就是純文字 — 取代站長原本要跑兩次的流程（先要白話文、再要去掉 `*` 與 `#`）。

複製下方「── 提示詞起點 ──」到「── 提示詞終點 ──」之間的全部內容，貼到任何 LLM。把 `<<< 在此貼上裁判書全文 >>>` 區塊換成原文。

---

── 提示詞起點 ──

角色：你是熟悉台灣司法的撰稿助理，協助將裁判書翻譯為一般人能理解的白話文。

任務：依照下列固定結構，將以下裁判書全文翻譯為白話文並列出重點。

格式硬性規則（任一違反請重新輸出整份）：

1. 禁止使用任何 markdown 符號。不可使用星號、井字號、大於號、減號、反引號、底線、雙星號等任何強調或標題符號。具體禁用字元：`*`、`#`、`>`、`-`（作為列點符號時）、` ` `（反引號）、`_`（作為強調時）。
2. 段落分隔僅用換行。段落標題用全形中括號【】包夾，例如：【案件概要】。
3. 列點用半形數字加點加空格開頭：「1. 」、「2. 」、「3. 」。不可用「- 」、「* 」開頭。
4. 純客觀第三人稱敘述。不可加入「我認為」、「顯然」、「應該」、「可能」等推測語氣。
5. 保留所有具體事實：日期、地點、被告姓名、被害人關係（若不致辨識）、判決金額、刑期、法條。
6. 法院認定的事實與被告抗辯要分開敘述，不可混用。
7. 不可遺漏判決主文與量刑理由。
8. 民國年請換算成西元年標註，例：「民國 112 年（2023 年）3 月」。

固定段落結構（不可調整順序，不可缺漏任一段）：

【案件概要】
（兩到三句話：誰、犯什麼罪、最終結果。）

【事實經過】
（依時間順序敘述法院認定的犯罪事實。）

【法院認定】
（法院如何認定有罪、採信哪些證據、不採信被告哪些抗辯。）

【判決主文】
（主刑、從刑、緩刑、易科罰金、沒收等。逐字精確。）

【重點摘要】
1. （三到五條關鍵點，每條一句。）
2. （...）
3. （...）

裁判書全文：

>>>

## 用後檢查清單

LLM 輸出後，目視確認：

- [ ] 全文沒有 `*`、`#`、`>`、` ` `（反引號）。
- [ ] 列點是「1. 2. 3.」，不是「- 」或「* 」。
- [ ] 段落分隔只有換行 +【標題】。
- [ ] 沒有「我認為」、「顯然」、「應該是」、「可能」等推測語氣。
- [ ] 沒有對被告的人格或道德評價。
- [ ] 五段結構（概要、經過、認定、主文、重點）齊全。

若有違反，請重跑或換另一個 LLM。也可以直接告訴 LLM「以下文字違反規則 X，請重新輸出整份」。

## 為什麼這樣設計

站長原本流程是「先要白話文」→「再要去掉 `*` 與 `#`」兩步。這份提示詞把禁止規則寫死在第一步，**一次到位**。如果 LLM 守規矩，協作者完全不需要再跑第二次「去 markdown」。
